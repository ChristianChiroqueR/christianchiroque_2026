Reproducible pipelines in R with targets

Why I stopped using numbered scripts and how targets transformed my political-analysis workflows.

For years I followed the classic convention: 01_clean.R, 02_model.R, 03_plots.R. It worked… until it didn't. One change in a data file, and I'd have to rerun everything from scratch — with no idea which steps were now stale.

The targets package solves this with a make-inspired approach: you declare a dependency graph and the tool only recomputes what changed.

Minimal structure

library(targets)
tar_option_set(packages = c("tidyverse", "tidymodels"))

list(
  tar_target(raw,    read_csv("data/raw.csv")),
  tar_target(clean,  clean_data(raw)),
  tar_target(model,  train(clean)),
  tar_target(report, render_report(model))
)

Why it changed my workflow

  • Zero ambiguity about what ran and when.
  • Automatic caching: if the data didn't change, the model isn't retrained.
  • DAG visualization via tar_visnetwork().

Where I use it today

Every academic and professional project of mine runs on targets. Especially useful for team collaborations where not everyone shares the same environment: the dependency graph documents the analysis better than any README.
